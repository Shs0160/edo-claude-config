# EDO 기능 카탈로그

각 기능의 의도와 현재 스펙을 누적. 새 기능 추가/변경 시 아래 템플릿 복사해 append.

---

## 템플릿

```
## [카테고리] 기능명

- **의도**: 왜 만들었는지 (어떤 문제를 해결하는지)
- **현재 스펙**: 현 시점의 핵심 동작 요약
- **최초 릴리즈**: I{n} (또는 "개발 중")
- **관련 Work Item**: #12345
- **관련 문서**: edo_docs 경로
- **메모**: 특이사항, 제약, 미해결 이슈
```

---

## [정책] 조건부 정책

- **의도**: 특정 조건(사용자, 문서 유형, 행위 등)에 따라 다른 집행 결정을 내릴 수 있도록
- **현재 스펙**: 등록/조회 UI, 목록 검색 필터, 상세 슬라이드 UI. JSON validation(문법/구조/스키마/비즈니스 로직) 포함
- **최초 릴리즈**: I15 이전
- **관련 문서**: `edo_docs/docs/06.internal/01.planning/01.release_note.md`

## [정책] 집행 정책

- **의도**: 조건부 정책 매칭 후 실제 수행할 동작을 정의
- **현재 스펙**: 결과(Result) 필드 정의, 노드 연결 구조, 집행 정책 명 / 상태 표시 / Flowchart UI
- **관련 릴리즈**: I15 (구조 정의), I16 (UI 최신화)

## [워크플로우] 집행 워크플로우 Iframe

- **의도**: 타 서비스 페이지 내에서 EDO 정책/매핑 관리를 할 수 있도록 iframe 기반 UI 제공
- **현재 스펙**: iframe UI, 부모창-iframe 데이터 연동, JSON 편집 슬라이드
- **최초 릴리즈**: I17 (개발 중)

## [템플릿] 템플릿 관리

- **의도**: 재사용 가능한 정책 구성 단위를 관리
- **현재 스펙**: 생성·조회·수정·복사
- **최초 릴리즈**: I17 (개발 중)

## [매핑] 정책-템플릿 매핑

- **의도**: 트리거/이벤트와 템플릿을 연결, 실행 시 우선순위에 따라 해석
- **현재 스펙**: 전사 공통 매핑 + 정책별 개별 매핑 2계층 구조
- **최초 릴리즈**: I17 (개발 중)

## [연동] 서비스 Integration

- **의도**: SHIELD Drive, SHIELD Gate 등 타 서비스와 EDO 정책 구조 연동
- **현재 스펙**: 설계 진행 중. 서비스 자체의 조건부 정책 구조와 EDO 정책 구조가 이중으로 존재하는 문제 식별됨
- **메모**: 관리 주체·연동 구조 결정 필요 → `memory/decisions.md` 참조
